安装cIOS ​

cIOS（自制系统）是一种自定义软件，由一些处理非官方内容的应用程序使用。 IOS 是控制台用来在硬件和软件之间进行通信的。 通过对官方 IOS 应用自定义补丁，可以运行更高级的自制软件，例如备份加载程序。

然而，有许多应用程序不需要 cIOS。 大部分Wiibrew上的Homebrew自制程序只需要 Homebrew Channel即可运行。

需要准备的东西

  • SD 卡（最好是非 SDHC）或 USB 设备

  • Homebrew Channel（必须是最新的！）

操作结果

  • 安装2个自制系统，可以使用备份加载器，一部分基于自制系统的应用也可以使用了。

指南 - 关于离线部分操作 ​

NUS离线下载 ​

如果你的Wii游戏机WiFi功能无法使用，或者网络状况不是很好，请先执行以下附加步骤。 否则跳到下面的指南。

WiiU 使用另一种方法来获取这些 WAD，因此如果您正在对 Wii U (vWii) 进行软改装，无论如何都应该跳过这些步骤。

不要在 Wii Mini 上执行这些步骤，否则您的主机会变砖！

在电脑上的操作

  1. 打开NUS Downloader，如果你没有这个程序，点我可以下载

  2. 请确认"Pack WAD"按钮已经被选中.

  3. 点击database...按钮，按下面顺序选择，每选择好一个，就点一下start NUS Download，一共会点两次start NUS Download：

  • IOS -> IOS56 -> v5661

  • IOS -> IOS57 -> v5918

  1. 打开NUS Downloader的文件夹，你会看到名为titles的文件夹。里面有两个文件夹：

  • 0000000100000038

  • 0000000100000039

每个文件夹中都有你下载的WAD文件，一共两个：

  • 0000000100000038 -> 5661 -> IOS56-64-v5661.wad

  • 0000000100000039 -> 5918 -> IOS57-64-v5918.wad

复制出这两个WAD文件，放到SD卡根目录中。准备工作就完成了。

如果希望验证下载的 WAD 是否正确，请使用MD5、SHA1工具（如 fciv）来验证：

  • IOS56

    • MD5 = 5cdee6593cf0dacc18cf300b12166fde

    • SHA1 = 597c360e521ccd3062fd9c38c5369e691344d5e2

  • IOS57

    • MD5 = ac8bbbea38f29e8d8959badb3badf18e

    • SHA1 = b8fd4efbb6d7ae2f4e9328b3082901f5981701b1

提示

如果你不确定你自己的操作是否正确，请一定要验证MD5！MD5不符合的话，你肯定就做错了。

操作指南 ​

在电脑上的操作

  1. 首先下载 cIOS 安装程序（确保下载正确的安装程序）：

  • Wii: 下载

  • Wii Mini: 本指南目前不提供任何Wii Mini相关的操作指引!!

  • WiiU (vWii): 下载

zip压缩包内将是一个 apps 文件夹。 将此文件夹放到到您的SD卡根目录（如果弹出提示是否合并，与现有的文件夹合并即可）。

您的SD卡结构应如下所示：

SD

|- IOS56-64-v5661.wad (可选，离线安装时需要)

|- IOS57-64-v5918.wad (可选，离线安装时需要)

|- /apps

    |- /d2x-cios-installer

在Wii游戏机上的操作

  1. 将 SD 卡插入 Wii。（不需要关机）

打开 Homebrew Channel 应用程序。

选择 d2x cIOS installer 并运行。

  1. 等待应用加载。 阅读欢迎屏幕上的文字（如果您感兴趣），然后按 A 继续。

  2. 现在可以看到一个配置页面。

安装 cIOS 时可以进行许多不同的配置。 本指南仅显示我们根据一般社区的经验提出的建议。 它们应该适用于 99% 的情况。

本指南将安装以下 cIOS：

  • cIOS 249 base 57 v10 beta52

  • cIOS 250 base 56 v10 beta52

安装方法:

变砖风险告知

请在文档未告知按下A键时千万不得擅自按下A键，否则可能变砖。

  • 你的游标指示会显示在左侧，请看清当前操作行后再操作，游标将显示为：( > )

  • 请选择第一行，将cIOS选项设置为v10 beta52。（按下手柄左右按键调整值）

  • 移动到下一行，将cIOS base选项设置为57。（按下手柄上下按键调整所选行）

  • 移动到下一行，将cIOS Slot选项设置为249。

    • 在某些情况下，它可能会警告您current (c)IOS 或 IOS STUB 将被覆盖。 您可以忽略此警告。

  • 移动到下一行，将cIOS revision选项设置为 65535。

    • cIOS revision仅供 Wii 在检查更新时使用。 它被设置为最高版本，以防止系统更新用官方 IOS 覆盖它。

    • 此选项不会出现在 WiiU(vWii) 安装程序中。

您现在已准备好安装，因此请按 A（第一次按下） 继续。 在 Wii 版本的安装程序中，您将看到可用的 cIOS 插槽的概览（对于 Wii U，请在下一个屏幕截图下方继续）

您刚刚选择的插槽应该闪烁。 其他几个插槽可能会以 红色 或 紫色 突出显示，具体取决于您的控制台和可能的以前的 softmod。

按 A（第二次按下） 开始安装。

等待进度完成。 这可能需要一段时间，尤其是对于在线安装。

完成后，您将看到另一个 IOS 概览屏幕。 您新安装的 cIOS 应该以绿色突出显示。

在 Wii U 版本上，您会收到一条简单的确认消息。

按 A（第三次按下，一轮安装结束） 按钮返回配置屏幕。

接下来，您要配置第二个 cIOS。 这次，使用以下设置：

  • d2x version: v10 beta52

  • cIOS base: 56

  • cIOS slot: 250.

  • cIOS revision: 65535

配置完成后，再次按 A 进行安装并等待该过程完成，下面两次按下A键不再赘述。

安装完所有内容后，按 B 退出应用程序。

恭喜你，完成了！

您现在可以运行任何您想要的备份加载程序！开始享受吧！

我们建议您使用USB Loader加载备份的游戏。

USB Loader不需要刻录 DVD，并且具有更好的速度和接口。

你可以点这里找到他们。

常见问题 ​

在执行此操作之前我需要安装任何东西吗？

你只需要安装一个最新的 Homebrew Channel，除此外不需要任何其他软件。

但是 Trucha Bug Restorer / Dop-Mii / IOS236 呢？

这些都是过时的软解，使用/安装它们毫无用处。

我什么时候应该按照“NUS离线下载”下的步骤进行操作？

这些步骤适用于Wii游戏机上无法使用WiFi，或网络状况不好的人使用。 这些步骤实质上是预先下载一些所需的文件，否则这些文件将由游戏机链接WiFi后自己下载。 由于法律原因，这些文件不能包含在安装程序中。

对 Wii U (vWii) 进行软改装的人无论如何都可以跳过这些步骤，因为安装程序会从控制台本身中提取它们。

我可以使用其他 cIOS 配置作为本指南中的配置吗？

是的！ 如上所述，本指南的配置是基于社区各种测试的建议。 他们应该适用于大多数情况。 但是你可以自由地在不同的插槽中安装不同的 cIOS。 不过，我们只建议在我们的配置不起作用时才这样做。 如果您确实安装了自己的 cIOS，请确保选择 200 以上的插槽。也不要使用插槽 254（即 BootMii IOS）。

如果还有 beta53，为什么还要安装 beta52？

beta52 总体上更稳定，并将提供比 beta53-alt 更高的兼容性。 但是，如果您愿意，可以选择在另一个插槽中安装 beta53 cIOS。 主要好处列在official D2X cIOS repository.

IOS base是做什么用的？

由于从头开始构建 IOS 并不容易； 我们安装的 cIOS 只是对任天堂为 Wii 提供的现有 IOS 打了一些补丁。 IOS 基础决定了补丁适用于哪些 IOS。

IOS slot有什么用？

Wii 能够在其内部存储器中存储多个 IOS，每个 IOS 都占用自己的插槽。 插槽只是一个数字，告诉系统从哪里加载 IOS。 例如，IOS56 将在 IOS 插槽 56 中找到。为了避免覆盖系统上安装的默认 IOS，cIOS（自定义 IOS，因此占用相同的插槽）安装在 200 以上的插槽中，它们默认都是空闲的空间。

IOS revision有什么用？

每个 IOS 都有一个内部版本号。 当需要对 IOS 进行更新时，版本号会增加。 系统在检测到 IOS 的更新版本时，将提示用户开始系统更新。 因此，已安装的 cIOS 的修订版通常会被最大化，以防止系统使用无法正常运行的官方版本覆盖它。

什么是NAND Emulation？

NAND Emulation是一项可选功能。 它允许您从假 NAND（控制台的内部存储器）加载虚拟控制台游戏等内容。 这种伪造的 NAND（通常称为 EmuNAND，Emulated NAND）存储在 SD 卡上而不是控制台本身。

USB Loader GX 和 Wiiflow 等 USB Loader 可以创建 EmuNAND 并使用它来启动游戏。

安装程序底部列出的"Current IOS"是什么？

此 IOS 是用于运行安装程序本身的 IOS（所有应用程序都需要一个 IOS 才能运行）。 它与安装的 cIOS 无关。

我需要安装 cIOS 才能使用 Nintendont (GameCube Backup Loader)吗?

不是，cIOS 不用于 GameCube 备份。 但是，Wii 备份需要它，一些 USB 加载器可能需要它才能运行。
